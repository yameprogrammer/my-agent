# 제품 범위와 요구사항

## 제품 정의

본 시스템은 한국형 장편 웹소설을 기획부터 집필까지 운영하는 내부 제작 시스템이다. 사용자는 작품 아이디어를 입력하거나 비워 둔 상태로 시작할 수 있으며, 시스템은 소재 제안, 주제 선정, 구조 설계, 아크 배치, 회차 설계, 집필, 검수를 단계적으로 수행한다.[cite:12][cite:15]

## 주요 사용자

- 총괄 기획자: 작품 방향과 시즌 전략을 승인한다.
- 스토리 디렉터: 메인 아크, 서브 아크, 복선 전략을 조정한다.
- 편집자: 화별 설계와 집필 결과를 검토한다.
- 운영자: 에이전트 실행과 상태 전이를 관리한다.
- AI Coding Agent: 본 문서를 바탕으로 시스템을 구현한다.

## 기능 범위

### 포함 범위

- 소재 및 주제 제안
- 작품 전체 설계
- 메인/서브 아크 설계
- 화별 사이클 설계
- 화별 세부 설계
- 장면 설계
- 실제 집필
- 연속성/설정 충돌 점검
- 복선 추적
- 작품 메모리 관리
- 재계획 및 개정

### 제외 범위

- 플랫폼 자동 업로드
- 표지 일러스트 생성
- 결제/상업 유통 기능
- 독자 실제 피드백 수집 기능

## 비기능 요구사항

- 재현성: 동일 입력과 동일 메모리 상태에서 동일 결과를 최대한 재현 가능해야 한다.
- 추적성: 어떤 문맥이 어떤 산출물 생성에 사용되었는지 로그가 남아야 한다.
- 개입 가능성: 인간 편집자가 각 단계에서 승인, 수정, 보류할 수 있어야 한다.
- 확장성: 장르별 템플릿과 모델 교체가 쉬워야 한다.
- 비용 통제: 상위 설계와 하위 집필 모델을 분리해 비용을 조절할 수 있어야 한다.

## 핵심 성공 지표

- 30화 이상 연속 생성 시 주요 설정 충돌 빈도
- 화말 훅 적합률
- 미회수 복선 추적 정확도
- 인물 성격 일관성 유지율
- 인간 편집자의 재작성 비율
- 독자 시뮬레이터 기준 다음 화 클릭 유도 점수
