# 프로젝트 개발 및 보안 규칙 (Project Rules)

본 프로젝트에 참여하는 모든 AI 에이전트는 다음 보안 및 개발 가이드라인을 철저히 준수해야 합니다.

---

## 🔒 1. 보안 및 크리덴셜(Secrets) 관리 규칙
1. **하드코딩 금지**: 
   - API Key (OpenAI 등), 데이터베이스 패스워드, JWT Secret Key 등 어떠한 비밀 정보도 소스 코드에 절대 하드코딩하지 마십시오.
   - 모든 보안 변수는 `.env` 파일과 `app/core/config.py` (Pydantic Settings)를 통해서만 관리되어야 합니다.
2. **Git 추적 제외 확인**:
   - 커밋하기 전, 민감한 설정 파일(`.env`), SQLite 데이터베이스 파일(`*.db`, `*.db-*`), 의존성 캐시(`.venv`) 등이 `.gitignore`에 등록되어 제외되었는지 반드시 검증하십시오.
3. **기본값 안전 조치**:
   - `config.py`에 중요 변수의 기본값(Default)을 명시할 경우, 배포 환경에서 반드시 덮어쓰여지도록 경고 처리하거나 로컬 개발용 임시값만 적용하십시오. (예: `JWT_SECRET` 기본값은 단순 개발용 더미로 지정하고 프로덕션 로딩 시 검증)

---

## 🏗️ 2. 코드 품질 및 비동기 준수 규칙
1. **Async Only**: 
   - LLM API 호출, 데이터베이스 연결, 외부 네트워크 요청은 반드시 `async/await` 비동기 패턴을 적용하여 갤럭시 Z 폴드 4 환경의 CPU 자원 지연을 방지하십시오.
2. **단위 테스트 필수**:
   - 신규 기능이 추가될 때마다 `tests/` 폴더 내에 단위 테스트(Unit Test)를 작성하여 정상 작동 여부를 선제 검증하십시오.
